# Media, Files, Products, and Selections

## Purpose

This document defines how ControlP.io projects and tasks connect to media, files, products, product selections, materials, proofs, artwork, and production assets.

## File and Media Types

Support:

- Photo
- Video
- Artwork
- Proof
- PDF
- Design file
- Production file
- Install photo
- Completion photo
- Customer upload
- Vendor upload
- Invoice
- Quote
- Contract
- SOW
- Spec sheet

## FAB Media Actions

The Gantt FAB/action dock should support:

- Upload photo
- Take photo
- Upload video
- Take video
- Upload file
- Attach existing file
- Add artwork
- Add proof
- Add completion photo

Where browser/device allows:

- Front-facing camera capture
- Rear-facing camera capture
- Mobile upload/capture workflows

## Attachment Targets

Files/media should attach to:

- Project
- Task
- Order
- Order item
- Production job
- Product selection
- Customer
- Vendor
- Message
- Proof approval

## File Visibility

Visibility options:

- Internal only
- Customer visible
- Vendor visible
- Public link, if supported
- Approval required
- Archived

## Product and Material Selections

Projects and tasks should support product/material selections.

Selection fields:

- Vendor
- Product name
- Category
- Manufacturer
- SKU/model
- Material type
- Print method
- Finish
- Size
- Color
- Image
- Gallery
- Video
- Price
- Quantity
- Subtotal
- Availability status
- Delivery status
- Delivery date
- Client approval status
- Internal approval status
- Production status
- Notes

## Product Selection Workflow

Example workflow:

1. Add product/material selection to project/task.
2. Attach vendor and product details.
3. Attach images/specs.
4. Request internal approval, if needed.
5. Request client approval, if customer-facing.
6. Track availability and delivery status.
7. Connect selection to order/production task.
8. Track completion and cost.

## Proof Workflow

Proofs should connect to:

- Project
- Task
- Order
- Customer
- Designer
- Approval status
- Comments/messages
- Version history

Proof status examples:

- Draft
- Ready for internal review
- Sent to customer
- Approved
- Rejected
- Changes requested
- Final approved

## AI Agent Integration

The AI Agent should be able to:

- Summarize attached files.
- Identify missing artwork.
- Identify pending proof approvals.
- Draft customer proof messages.
- List product selections needing approval.
- Flag unavailable materials.
- Summarize production assets for a task.
